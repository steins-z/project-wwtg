## 一、产品概述

- **定位：** 对话式 AI 助手，30 秒内根据位置/人数/状态生成个性化周末出行方案
- **目标用户：** 一二线城市 20-35 岁年轻人
- **MVP 覆盖城市：** 苏州 + 上海 + 杭州
- **交付形态：** 微信小程序（评审决议，原 PRD 为独立 APP）

### 核心流程

用户发起对话 → Agent 最多问 3 个问题（城市/人数/状态）→ 15 秒内出 2 个方案卡片 → 用户选"就这个"

### 成功指标

- 方案采纳率 > 40%
- 对话轮数 < 5 轮
- 周回访率 > 30%

## 二、用户痛点

| 痛点 | 典型用户声音 |
| --- | --- |
| 🧠 决策瘫痪 | "刷了很多攻略，最后决定躺一天" |
| 📍 地域不匹配 | "可是我在小县城" |
| 😴 精力错配 | "上班想出去，周末只想躺" |
| 🧩 信息碎片化 | "要多久？能推车吗？有停车场吗？" |
| 🎭 场景不匹配 | 独处/约会/亲子需求完全不同 |

### 用户画像

| 人群 | 核心痛点 |
| --- | --- |
| 独居打工人 | 周末不知道干嘛，刷手机一天就没了 |
| 情侣 | 约会没新意，想找互动性强的活动 |
| 带娃家长 | 需要亲子友好、安全、低消费的去处 |
| 小城市用户 | "可是我在小县城"——热门攻略全是大城市的 |

## 三、功能清单（MVP Scope）

### P0 必须有

| 功能 | 描述 |
| --- | --- |
| 智能提问 | 最多3个问题收集上下文，支持一句话多信息解析 |
| 天气查询 | 自动查目标城市周末天气，影响推荐 |
| 方案生成 | 基于小红书数据，输出2个方案卡片 |
| 方案详情 | 选定后展开：路线站点、时间、花费、tips |
| 地图导航 | 每个站点提供导航跳转链接（高德/百度） |
| 拒绝重推 | 不喜欢可以说原因，换一批 |
| 特殊标签 | 孕妇友好/亲子/无障碍/免费等标签筛选 |

### P1 锦上添花

| 功能 | 描述 |
| --- | --- |
| 分享卡片 | 生成图片卡片，可分享给朋友/伴侣 |
| 保存日历 | 一键添加到系统日历/提醒 |
| 查看原帖 | 链接到小红书原帖 |
| 天气提醒 | 出发前2小时推送天气变化提醒 |

### P2/P3 后续迭代

偏好学习、周五推送、费用估算、拼团约伴、行后反馈、多城市支持

## 四、方案卡片信息结构

```
方案卡片 {
  标题: "双塔市集赏花散步"
  一句话描述: "玉兰花季，吃喝逛一条线"
  emoji图标: 🌸
  基础信息 { 预计时长, 人均花费(粗粒度区间), 交通方式, 适合人数, 距离市中心 }
  标签[] { ✅孕妇友好, ✅人少, ✅免费, ✅有餐饮 }
  路线站点[] {
    站点 { 名称, 建议到达时间, 停留时间, 推荐内容, 导航链接, 步行到下一站距离 }
  }
  贴心提示[] { 天气穿衣, 路况, 卫生间位置 }
  数据来源[] { 帖子标题, 赞数, 链接 }
}
```

## 五、对话设计规范

**语气：** 像靠谱的本地朋友，简洁有趣，emoji 适度

**示例：**

- 标准流程："周末好！📍在哪？👥几人？⚡想动还是躺？"
- 一句话跳过："苏州，带3岁娃，人少公园" → 直接出方案
- 拒绝后追问原因，调整重推

**禁止：** ❌ 超过2个方案 ❌ 长篇大论 ❌ 商业广告 ❌ 恶劣天气推户外 ❌ 催促决定

### 性能优化方向

| 环节 | 当前耗时（估） | 优化方向 |
| --- | --- | --- |
| 小红书实时搜索 + 抓帖子内容 | ~90s | 每日预缓存，查询时直接读缓存 |
| 天气 API 查询 | ~5s | 可并行，问题不大 |
| 地理编码/距离计算 | ~10s | 可并行 |
| LLM 生成方案文案（串行调 2 次） | ~60s | 并行生成 2 个方案 + streaming |
| 整体串行等待 | ~剩余 | 所有 API 并行化 |
